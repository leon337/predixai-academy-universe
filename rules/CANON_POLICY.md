# Política de Cânone

## 1. Informação canônica

Uma informação torna-se canônica somente quando:

1. foi discutida;
2. foi aprovada explicitamente;
3. foi registrada neste repositório;
4. aparece no estado oficial ou no documento correspondente.

Informações canônicas devem ser respeitadas por aulas, cursos, agentes de IA, exemplos e projetos integradores.

## 2. Informação não canônica

São não canônicos:

- rascunhos;
- propostas;
- alternativas ainda em análise;
- testes narrativos;
- hipóteses;
- exemplos temporários.

Conteúdo não canônico deve ser identificado com o marcador `STATUS: NÃO CANÔNICO`.

## 3. Alterações

Uma informação canônica não deve ser contradita, removida ou reescrita silenciosamente. Mudanças exigem:

1. justificativa;
2. análise de impacto;
3. aprovação explícita;
4. atualização do `CHANGELOG.md`;
5. revisão dos conteúdos dependentes.

## 4. Conflitos

Quando duas fontes divergirem, prevalece esta ordem:

1. `rules/CANON_POLICY.md`;
2. `UNIVERSE_STATE.md`;
3. documentos canônicos específicos;
4. `TIMELINE.md`;
5. conteúdo de cursos;
6. conversa atual.
